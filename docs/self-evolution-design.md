## 一、核心理念

> "The agent that grows with you"（与你一起成长的智能体）

**设计哲学**：不是靠更大的上下文窗口，而是靠**跨会话持久化**和**主动学习机制**，让 Agent 越用越强。

**痛点解决**：
- 传统 Agent "健忘"：每次对话从零开始
- 能力固定：不会从经验中学习
- 需反复调教：同一任务重复推理

## 二、学习闭环流程

```
┌─────────────────────────────────────────────────────────────┐
│                    Hermes 学习闭环                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   执行任务 ──► 自我评估 ──► Skill 创建 ──► 技能沉淀          │
│      │            │              │              │            │
│      │       (每15次工具调用)    │              │            │
│      │            │              ▼              │            │
│      │            │        写入 Skill 库        │            │
│      │            │              │              │            │
│      │            ▼              │              │            │
│      │     "什么有效？什么失败？" │              │            │
│      │            │              │              │            │
│      │            └──────────────┘              │            │
│      │                    │                      │            │
│      ▼                    ▼                      ▼            │
│   下次调用 ───────────────────────────────► 直接复用 Skill     │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 关键触发条件

| 触发点 | 条件 | 动作 |
|--------|------|------|
| 自我评估 | 每 15 次工具调用 | 暂停反思：什么有效？什么失败？ |
| Skill 创建 | 复杂任务完成后 | 自动生成可复用的 Skill 文件 |
| 技能改进 | 使用 Skill 时发现失败 | 通过 GEPA 算法优化 |

## 三、六大子系统架构

```
┌────────────────────────────────────────────────────────────────┐
│                      Hermes Agent 架构                          │
├────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐         │
│  │ 1. 周期性   │    │ 2. 后台     │    │ 3. 双文件   │         │
│  │    Nudge    │───►│   Review    │───►│    存储     │         │
│  │ (每10 turn) │    │ (异步复盘)  │    │ (MEMORY.md) │         │
│  └─────────────┘    └─────────────┘    └─────────────┘         │
│         │                  │                   │               │
│         ▼                  ▼                   ▼               │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐         │
│  │ 4. 自主     │    │ 5. Memory   │    │ 6. 全息     │         │
│  │    Skill    │    │   Manager   │    │    Memory   │         │
│  │    创建     │    │  (编排层)   │    │  (向量检索) │         │
│  └─────────────┘    └─────────────┘    └─────────────┘         │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

### 子系统详解

| 子系统 | 功能 | 技术实现 |
|--------|------|----------|
| **周期性 Nudge** | 触发学习信号 | 每 10 turn 自动提醒"该学习" |
| **后台 Review** | 异步复盘 | daemon 线程，不阻塞主流程 |
| **双文件存储** | 持久化知识 | MEMORY.md + USER.md |
| **自主 Skill 创建** | 封装成功操作 | `create_skill` 工具 + 原子写入 |
| **Memory Manager** | 统一接口 | 支持 Pinecone/Weaviate 插件 |
| **全息 Memory** | 智能检索 | SQLite FTS5 + HRR 向量 |

## 四、四层内存设计

```
┌────────────────────────────────────────────────────────────┐
│                    四层内存架构                              │
├────────────────────────────────────────────────────────────┤
│                                                             │
│  L1: Frozen Snapshot (最快)                                │
│      └── 会话启动时生成不可变快照                            │
│          └── 稳定 prompt cache 命中率                       │
│                                                             │
│  L2: MEMORY.md / USER.md (快)                              │
│      └── 长期身份认知、用户偏好                              │
│          └── 跨会话持久化                                   │
│                                                             │
│  L3: Holographic Memory (中)                               │
│      └── SQLite + FTS5 全文索引                             │
│          └── HRR 向量检索 + 信任评分                        │
│                                                             │
│  L4: 外部知识库/API (慢)                                    │
│      └── 领域知识、实时数据                                 │
│          └── MCP 扩展                                       │
│                                                             │
└────────────────────────────────────────────────────────────┘
```

| 层级 | 存储方式 | 速度 | 适用场景 |
|------|----------|------|----------|
| **L1** | Frozen Snapshot | 最快 | 会话初始化 |
| **L2** | MEMORY.md/USER.md | 快 | 身份认知 |
| **L3** | Holographic Memory | 中 | 知识检索 |
| **L4** | 外部知识库 | 慢 | 领域知识 |

## 五、GEPA 自我进化算法

> GEPA = Genetic-Pareto Prompt Evolution（遗传帕累托提示进化）

### 算法流程

```
┌─────────────────────────────────────────────────────────────┐
│                    GEPA 进化流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  读取当前 Skill/Prompt                                       │
│         │                                                    │
│         ▼                                                    │
│  生成评估数据集 ←── 执行轨迹 (Execution Traces)              │
│         │                                                    │
│         ▼                                                    │
│  GEPA 优化器 ─────► 生成候选变体                             │
│         │                    │                              │
│         │                    ▼                              │
│         │              评估变体效果                           │
│         │                    │                              │
│         │                    ▼                              │
│         │              约束门控检查                          │
│         │                    │                              │
│         │                    ▼                              │
│         └──────────────► 最优变体 ──► 提交 PR               │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 核心特点

| 特点 | 说明 |
|------|------|
| **反思性变异** | 分析执行轨迹，理解失败原因，针对性改进 |
| **帕累托前沿** | 保持多样性，不陷入局部最优 |
| **自然语言反馈** | 替代传统数值奖励，更适合 LLM |

### 效率提升

| 指标 | 对比传统 RL |
|------|-------------|
| 进化效率 | 提升 40% |
| 样本利用率 | 提高 65% |

## 六、约束门控（进化验证）

**所有进化变体必须通过约束门控**：

```
┌─────────────────────────────────────────────────────────────┐
│                    约束门控检查                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 完整测试套件                                             │

│  2. 尺寸限制                                                 │

│     └── 不能偏离原始目的                                     │
│                                                              │

│     └── 所有变更需人工确认                                   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

**安全闭环**：自动生成 → 人工确认 → PR 审核

## 七、Skill 自动生成机制

### 触发条件

| 触发场景 | 条件 |
|----------|------|
| 复杂任务完成 | 5+ 工具调用 |
| 重复模式识别 | 相似操作执行 3 次+ |
| 用户明确请求 | `/create-skill` 命令 |

### Skill 结构

```
skill-name/
├── SKILL.md          # 技能描述（Markdown，人类可读）
├── metadata.yaml     # 元数据
└── scripts/          # 执行脚本（可选）
```

**遵循标准**：agentskills.io 开放标准

### 生成流程

```
任务执行
    │
    ▼
检测可复用模式
    │
    ▼
调用 create_skill 工具
    │
    ▼
生成 SKILL.md 文件
    │
    ▼
写入技能库
    │
    ▼
下次直接调用
```

## 九、设计方案（适配 Claude Code）

### 设计原则

1. **轻量级**：不引入复杂依赖
2. **渐进式**：先验证价值，再逐步增强
3. **安全优先**：人工审核必须

### Phase 1：最小可行方案

```yaml
# 建议实现

触发条件:
  - 完成复杂任务（5+ 工具调用）
  - 用户明确请求

动作:
  1. 分析当前对话 transcript
  2. 提取可复用模式
  3. 生成 Skill 草稿
  4. 请求用户确认
  5. 写入 ~/.claude/skills/

验证:
  - 用户确认（必须）
  - 简单测试用例
```

### Phase 2：自动改进

```yaml
# 进阶能力

触发条件:
  - Skill 使用失败
  - 用户反馈不佳

动作:
  1. 记录失败场景
  2. 分析失败原因
  3. 生成改进版本
  4. A/B 测试对比
  5. 人工确认后更新

验证:
  - 对比测试
  - 用户评分
  - 收敛检查
```

### Phase 3：主动学习

```yaml
# 完整闭环

触发条件:
  - 周期性（每日/每周）
  - 检测到重复模式

动作:
  1. 扫描历史对话
  2. 发现优化机会
  3. 主动提议学习
  4. 用户确认后执行

验证:
  - 完整测试套件
  - 回归测试
  - 效果对比报告
```
