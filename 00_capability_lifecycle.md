# 能力生命周期

> 识别能力成熟度，动态调整配置策略

## 核心认知

AI 能力是动态演进的：

```
时间轴：
过去（需自己实现） ──────────────────────> 未来（Agent 内置）
     ↑                                        ↑
  skill/rules/hook 补充              直接用原生能力
```

**原则**：配置要精简，避免为已成熟能力维护低价值定制。

---

## 能力分层模型

| 层级 | 定义 | 处理方式 | 示例 |
|------|------|----------|------|
| **原生能力** | Agent 内置，开箱即用 | 直接使用 | 代码编辑、文件搜索、基础推理 |
| **扩展能力** | Agent 可做但需引导 | skill 定义流程 | TDD 流程、安全审查 |
| **外部能力** | Agent 原生不具备 | MCP + 外部工具 | 视频解析、企业系统 |
| **约束能力** | 限制 Agent 行为 | rules 定义边界 | 重构红线、格式要求 |
| **自动化能力** | 事件驱动执行 | hook 自动触发 | 提交前检查 |

---

## 成熟度判断清单

判断能力是否已成熟（可直接使用）：

```yaml
原生能力评估:
  - Agent 能独立完成？✓
  - 需要多轮对话纠正？✗（说明不成熟）
  - 需要外部工具？✗（说明是外部能力）

扩展能力评估:
  - 有明确可复用流程？✓
  - 通过 skill 可稳定触发？✓
  - 流程变化频率低？✓
```

### 判断示例

| 能力 | 判断 | 依据 |
|------|------|------|
| 代码重构 | 原生 | Claude 内置重构能力，无需额外配置 |
| TDD 流程 | 扩展 | 需要流程引导，skill 更稳定 |
| 视频解析 | 外部 | 需要 Whisper/API，通过 MCP 桥接 |
| 禁止硬编码密钥 | 约束 | rules 强制，不依赖 Agent 判断 |

## 配置兼容性

换 Agent 时的迁移成本：

| 配置类型 | 兼容性 | 迁移成本 | 建议 |
|----------|--------|----------|------|
| Rules | 高（纯文本） | 低 | 优先沉淀 |
| Skills | 中（需改格式） | 中 | 核心场景保留 |
| Hooks | 低（平台相关） | 高 | 谨慎依赖 |
| MCP | 中（协议标准化） | 中 | 优先标准协议 |
