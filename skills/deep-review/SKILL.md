# Deep Review

深度多维度代码审查，使用并行 subagents 全面分析代码。

---

## 与原生 /review 的区别

| 功能 | 原生 /review | /deep-review |
|------|--------------|--------------|
| 审查维度 | 单一维度 | 4维度并行（安全+质量+性能+架构） |
| 执行方式 | 串行 | 并行 subagents |
| 交互性 | 有 | 增强交互（分维度确认） |
| 输出格式 | 标准 | 分维度报告 + 汇总 |
| 修复建议 | 有 | 有 + 自动生成代码 |

---

## 四维度审查内容

### 1. 安全审查

| 检查项 | 说明 |
|--------|------|
| **注入漏洞** | SQL/Command/Code Injection |
| **认证授权** | Auth bypass, 权限控制 |
| **敏感数据** | 密码/API Key 硬编码 |
| **XSS/CSRF** | 跨站脚本/请求伪造 |
| **输入验证** | 用户输入过滤 |

### 2. 质量审查

| 检查项 | 说明 |
|--------|------|
| **SOLID 原则** | 单一职责、开闭原则等 |
| **代码复杂度** | 圈复杂度、嵌套深度 |
| **命名规范** | 变量/函数命名 |
| **重复代码** | DRY 原则 |
| **测试覆盖** | 缺失测试 |

### 3. 性能审查

| 检查项 | 说明 |
|--------|------|
| **算法复杂度** | O(n²) 以上警告 |
| **资源泄漏** | 未关闭连接/文件 |
| **循环优化** | 循环内重复计算 |
| **缓存机会** | 可缓存未缓存 |
| **N+1 查询** | 数据库查询问题 |

### 4. 架构审查

| 检查项 | 说明 |
|--------|------|
| **依赖关系** | 循环依赖、过耦合 |
| **模块边界** | 职责划分不清 |
| **设计模式** | 模式滥用/误用 |
| **扩展性** | 难以扩展的设计 |
| **一致性** | 风格不统一 |

## 使用方式

### 基本使用

```bash
# 审查当前变更（git diff）
/deep-review

# 审查指定文件
/deep-review src/main.py

# 审查目录
/deep-review src/
```

### 交互流程

```
1. 分析变更范围
   ├── 文件数: 5
   ├── 变更行数: 120
   └── 风险级别: 中

2. 并行执行 4 维度审查...

3. 分维度展示结果：

📊 安全审查（发现 2 个问题）
┌────────────────────────────────────────────┐
│ 🔴 P0: SQL注入风险                          │
│    文件: src/db.py:45                       │
│    问题: 用户输入直接拼接到 SQL                 │
│    修复: 使用参数化查询                      │
└────────────────────────────────────────────┘

是否修复这些问题？[y/n/select]

📊 质量审查（发现 1 个问题）
...

4. 用户选择修复维度

5. 生成修复代码并应用
```

```markdown
## 🔴 严重问题

### 1. SQL注入风险

| 属性 | 值 |
|------|-----|
| 文件 | src/db.py:45 |
| 维度 | 安全 |
| 严重性 | P0 |

**问题描述**：
用户输入直接拼接到 SQL 查询中。

**修复代码**：

    # 修复前
    query = f"SELECT * FROM users WHERE id = {user_id}"

    # 修复后
    query = "SELECT * FROM users WHERE id = ?"
    cursor.execute(query, (user_id,))

## ✅ 已确认修复

- [x] SQL注入风险 - 已应用参数化查询
- [ ] 密码硬编码 - 待处理
```

## 与原生 review 的集成

可以在原生 `/review` 后执行 `/deep-review` 获得更全面分析：

```bash
# 先用原生 review
/review

# 再用 deep-review 并行深查
/deep-review
```

## 配置选项

可在 `$HOME/.claude/config/deep-review.yaml` 配置：

```yaml
# 审查级别
level: "standard"  # quick | standard | deep

# 启用的维度
dimensions:
  security: true
  quality: true
  performance: true
  architecture: true

# 并发数
concurrency: 4

# 自动修复
auto_fix: false  # true 则不询问直接修复

# 忽略规则
ignore:
  - "tests/**"
  - "**/*.md"
```
